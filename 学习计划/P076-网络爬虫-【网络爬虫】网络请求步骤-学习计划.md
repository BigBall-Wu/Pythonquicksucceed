## 五、具体学习步骤

### 第 1 步：HTTP 请求的 7 步流程（15 分钟）

```python
# 一次完整 HTTP 请求的步骤：

# 1. DNS 解析：域名 → IP
#    example.com → 93.184.216.34
#    用 dig 或 nslookup 可以查看
import socket
ip = socket.gethostbyname("example.com")
print(ip)  # 93.184.216.34

# 2. TCP 握手：三次握手建立连接
#    客户端 → SYN → 服务器
#    服务器 → SYN+ACK → 客户端
#    客户端 → ACK → 服务器
#    握手延迟约 1 个 RTT

# 3. TLS 握手（HTTPS 才有）：
#    协商加密算法
#    验证证书
#    生成会话密钥
#    TLS 1.2 握手约 2 个 RTT
#    TLS 1.3 握手约 1 个 RTT

# 4. 发送 HTTP 请求
#    GET /path HTTP/1.1
#    Host: example.com
#    ...

# 5. 服务器处理
#    路由、分发、处理逻辑

# 6. 返回 HTTP 响应
#    HTTP/1.1 200 OK
#    Content-Type: text/html
#    ...
#    <html>...</html>

# 7. 浏览器渲染
#    解析 HTML
#    加载 CSS/JS
#    渲染 DOM
#    执行 JS
```

### 第 2 步：HTTP vs HTTPS（10 分钟）

```python
# HTTP vs HTTPS 对比

HTTP vs HTTPS = """
┌─────────────┬────────────────┬────────────────┐
│             │     HTTP       │     HTTPS      │
├─────────────┼────────────────┼────────────────┤
│ 端口        │ 80             │ 443            │
│ 加密        │ 无             │ TLS/SSL        │
│ 性能        │ 快（无握手）    │ 慢（TLS 握手） │
│ 证书        │ 不需要         │ 需要 CA 证书   │
│ 隐私        │ 不安全         │ 安全           │
│ SEO         │ 低             │ 高（Google 加分）│
└─────────────┴────────────────┴────────────────┘
"""

# 实际影响：
# - HTTPS 首次连接慢约 200-500ms（TLS 握手）
# - 但数据传输是加密的，更安全
# - 现代浏览器强制 HTTPS（Chrome 标记 HTTP 为不安全）

# 性能优化：
# - TLS 1.3 比 1.2 快约 40%
# - 使用 Session Ticket 复用会话
# - 启用 HTTP/2 或 HTTP/3
```

### 第 3 步：浏览器开发者工具观察请求（15 分钟）

```python
# 打开浏览器 F12 → Network 面板，观察请求：

# 1. 观察请求头（Headers）
#    Request URL: 请求的完整 URL
#    Request Method: GET/POST/PUT/DELETE
#    Status Code: 200/301/404/500
#    Remote Address: 服务器 IP:端口

# 2. 观察响应头
#    Content-Type: text/html; charset=utf-8
#    Content-Length: 12345
#    Set-Cookie: session=xxx; Path=/

# 3. 观察 Timing
#    DNS: DNS 解析耗时
#    Connect: TCP 握手耗时
#    TLS: TLS 握手耗时
#    TTFB: Time To First Byte（首字节时间）
#    Content Download: 内容下载耗时

# 4. 常见状态码
#    200 OK - 成功
#    301 Moved Permanently - 永久重定向
#    302 Found - 临时重定向
#    304 Not Modified - 缓存未修改
#    400 Bad Request - 请求错误
#    401 Unauthorized - 需要认证
#    403 Forbidden - 禁止访问
#    404 Not Found - 资源不存在
#    500 Internal Server Error - 服务器错误
```

### 第 4 步：Python 启用详细日志（15 分钟）

```python
import requests
import logging

# 启用 requests 详细日志
# 设置日志级别为 DEBUG
logging.basicConfig(level=logging.DEBUG)

# 或者只打印 HTTP 相关的日志
requests_log = logging.getLogger("urllib3")
requests_log.setLevel(logging.DEBUG)
requests_log.propagate = True

# 发送请求，会打印详细的请求和响应信息
response = requests.get("https://httpbin.org/get")

# 输出示例：
# DEBUG:urllib3.connectionpool->Starting new HTTPS connection (1): httpbin.org:443
# DEBUG:urllib3.connectionpool->https://httpbin.org:443 "GET /get HTTP/1.1" 200 306
```

### 第 5 步：测量请求耗时（10 分钟）

```python
import requests
import time

# 方法 1：用 time.time()
start = time.time()
response = requests.get("https://httpbin.org/get")
elapsed = time.time() - start
print(f"总耗时: {elapsed:.3f}秒")

# 方法 2：用 response.elapsed（推荐）
response = requests.get("https://httpbin.org/get")
print(f"请求耗时: {response.elapsed.total_seconds():.3f}秒")
print(f"状态码: {response.status_code}")

# 方法 3：测量各阶段耗时
def measure_request(url):
    """测量请求各阶段耗时"""
    # DNS 解析
    import socket
    start = time.time()
    domain = url.split("/")[2]
    socket.gethostbyname(domain)
    dns_time = time.time() - start

    # 连接 + 请求
    start = time.time()
    response = requests.get(url)
    total_time = time.time() - start

    return {
        "dns": dns_time,
        "total": total_time,
        "server": response.elapsed.total_seconds()
    }

result = measure_request("https://httpbin.org/get")
print(f"DNS: {result['dns']:.3f}秒, 总计: {result['total']:.3f}秒")
```
