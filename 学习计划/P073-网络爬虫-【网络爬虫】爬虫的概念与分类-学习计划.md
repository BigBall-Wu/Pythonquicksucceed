## 五、具体学习步骤

### 第 1 步：爬虫定义（5 分钟）

```python
# 网络爬虫：自动从网页提取数据的程序
# 三步骤：
# 1. 拿数据（HTTP 请求）
# 2. 解析数据（HTML/JSON 解析）
# 3. 保存数据（文件/数据库）

# 一个最简单的爬虫
import requests
r = requests.get("https://api.github.com")
data = r.json()                   # 1+2+3 一气呵成
```

### 第 2 步：按主题分类（10 分钟）

```python
# 通用爬虫：抓全网，搜索引擎用
# 例子：Googlebot
# 特点：覆盖面广，技术要求高

# 聚焦爬虫：抓特定主题/网站
# 例子：抓取某电商网站所有商品价格
# 特点：目标明确，业务导向

# 增量爬虫：只抓新内容
# 例子：新闻订阅（每小时只抓新发布的）
# 特点：基于时间戳或版本号

# 深度爬虫：跟踪链接深入抓取
# 例子：学术论文（从首页开始抓所有引用）
# 特点：需要去重防环
```

### 第 3 步：按实现技术分类（10 分钟）

| 方式 | 适用 | 性能 | 难度 |
|------|------|------|------|
| `requests + BeautifulSoup` | 简单静态页面 | 中 | 低 |
| `requests + lxml/XPath` | 高效解析 | 中 | 中 |
| `Scrapy` | 大型项目 | 高 | 高 |
| `Selenium` | JS 渲染 | 慢 | 中 |
| `Playwright` | 现代浏览器自动化 | 较快 | 中 |
| `pyppeteer` | 无头 Chrome | 中 | 中 |
| `Appium` | 移动 App | 慢 | 高 |

**选型决策树**：
```
页面需要 JS 渲染？
├─ 是 → Selenium / Playwright
└─ 否
    ├─ 项目复杂度？
    │   ├─ 小型 → requests + BeautifulSoup
    │   └─ 大型 → Scrapy
    └─ 反爬严重程度？
        ├─ 轻微 → requests
        └─ 严重 → Scrapy + 代理池 + 浏览器指纹
```

### 第 4 步：按行业应用分类（10 分钟）

| 行业 | 应用 |
|------|------|

### 第 5 步：实战 - 选型决策（10 分钟）

```python
# 项目 1：抓取豆瓣 Top 250 电影
# 决策树：
# - 静态 HTML？ ✓ → requests + BeautifulSoup
# - 翻页？ 是 → 循环 + URL 参数
# - 反爬？ 弱 → 加 User-Agent

# 项目 2：抓取知乎动态内容
# 决策树：
# - JS 渲染？ ✓ → Selenium 或 API 反代
# - 登录？ 是 → 模拟登录
# - 反爬？ 强 → 代理池 + 浏览器指纹

# 项目 3：电商价格监控（千万商品）
# 决策树：
# - 大规模？ ✓ → Scrapy
# - 分布式？ 考虑 → scrapy-redis
# - 反爬？ 中 → 限速 + 代理池
```

### 第 6 步：常见错误（10 分钟）

```python
# 错误 1：选错工具
# ❌ 静态页面用 Selenium（慢、复杂）
# ❌ 动态页面用 requests（拿不到内容）

# 错误 2：忽视法律
# ❌ 爬取受版权保护的内容
# ❌ 绕过付费墙
# ❌ 违反 robots.txt

# 错误 3：技术至上
# ❌ 不评估数据价值就开爬
# ❌ 不考虑存储成本（爬了 10TB 没地方放）

# 错误 4：忽视伦理
# ❌ 爬取个人隐私
# ❌ 频繁请求导致服务器宕机
# ❌ 出售/共享爬取的数据
```
